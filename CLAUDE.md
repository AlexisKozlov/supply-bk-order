# CLAUDE.md

Этот файл даёт рабочий контекст для Claude/Codex в этом репозитории.

## Язык

Проект на русском языке. Всегда отвечай на русском.

## Стиль общения

Пользователь не программист. Все пояснения, описания изменений и сводки пиши простым понятным языком, без лишнего жаргона. Коротко и по делу.

## Порядок работы

Перед любыми изменениями:

1. Сначала найди релевантные файлы.
2. Объясни, где именно будет реализована задача.
3. Предложи короткий пошаговый план и ожидаемую стратегию реализации.
4. Не вноси изменения, пока не показал этот план пользователю.

## Команды

```bash
npm run dev      # Vite dev-сервер фронтенда
npm run build    # Сборка фронтенда в dist/ и копирование PHP API в dist/api/
npm run preview  # Превью production-сборки
```

Тестов и линтера в проекте нет. Бэкенд — PHP, работает через веб-сервер, не запускается через `npm`.

## Архитектура

Стек:

- Vue 3 (Composition API, `<script setup>`) + Pinia + Vue Router + Vite + PWA
- PHP + MariaDB/MySQL
- Без TypeScript

Точки входа:

- Фронтенд: `index.html` -> `src/main.js` -> `src/App.vue` -> `src/router/index.js`
- Бэкенд API: `api/index.php`

Важно:

- Не опирайся на старые описания с корневым `index.php`: в актуальном репозитории основная backend-точка входа — `api/index.php`.

## Структура проекта

### Фронтенд

- `src/views/` — страницы и крупные модули интерфейса
- `src/stores/` — состояние и клиентская бизнес-логика
- `src/lib/` — расчёты, API-клиент, импорт/экспорт, утилиты
- `src/components/` — переиспользуемые компоненты
- `src/layouts/` — каркас приложения

Ключевые модули:

- `src/stores/userStore.js` — авторизация, сессия, роли, права, скрытые модули, техработы
- `src/stores/orderStore.js` — основной заказ, позиции, undo/redo, аудит
- `src/lib/apiClient.js` — REST/RPC-клиент
- `src/lib/calculations.js` — формулы расчёта заказа и буферного режима
- `src/lib/saveOrder.js` — сохранение/обновление заказа, diff, уведомления
- `src/lib/legalEntities.js` — логика юрлиц и групп

### Бэкенд

- `api/index.php` — главный роутер API
- `api/includes/helpers.php` — авторизация, RBAC, фильтры, общие функции
- `api/includes/rpc.php` — RPC-эндпоинты и большая часть серверной бизнес-логики
- `api/includes/crud.php` — REST CRUD для таблиц
- `api/includes/restaurant_orders.php` — кабинет и заказы ресторанов
- `api/includes/supplier_orders.php` — заявки поставщикам
- `api/includes/truck_loading.php` — загрузка машин
- `api/includes/legal_entities.php` — серверная логика групп юрлиц

## Юрлица

Есть две группы:

- `BK_VM` — `ООО "Бургер БК"` и `ООО "Воглия Матта"`
- `PS` — `ООО "Пицца Стар"`

Важно:

- Эта логика есть и на фронтенде, и на бэкенде.
- Перед изменениями, связанными с юрлицами, фильтрами, поставщиками, ресторанами или доступами, проверяй обе стороны:
  - `src/lib/legalEntities.js`
  - `api/includes/legal_entities.php`

## Права доступа

Права и роли распределены между несколькими слоями:

- `src/stores/userStore.js`
- `src/router/index.js`
- `api/includes/helpers.php`
- `api/includes/crud.php`
- местами `api/includes/rpc.php`

Если задача касается ролей, доступа к страницам, модулей, таблиц или действий пользователя, проверяй и фронтенд, и бэкенд.

## Миграции и данные

- Основные миграции лежат в `migrations/`
- Старые миграции и SQL-файлы для сборки есть в `api/migrations/`
- Есть отдельные SQL-заготовки в `sql/`
- Миграции применяются вручную

## Что не редактировать как исходники

- `dist/`
- `public/assets/`
- собранный `public/index.html`

Изменения вносить в исходники: `src/`, `api/`, `migrations/`, `scripts/`.

## Зоны риска

- `api/includes/rpc.php` — большой монолитный файл, здесь легко задеть соседнюю логику
- логика юрлиц продублирована между фронтом и бэком
- RBAC и проверки доступа находятся в нескольких местах
- из-за отсутствия тестов любые изменения нужно проверять вручную особенно аккуратно
