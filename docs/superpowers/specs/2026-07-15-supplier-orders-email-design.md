# Этап 1 — Почта поставщику (модуль «Заявки поставщикам» / `/supplier-orders`)

Дата: 2026-07-15
Статус: утверждён к реализации

## Контекст

Модуль `/supplier-orders` (so_*) позволяет ресторанам подавать заявки напрямую
поставщикам, а закупщику — видеть сводку и выгружать Excel. Сейчас сводку можно
отправить поставщику **только в Telegram** (`POST so/admin/send-summary`). Отправки
по email нет, хотя это самый частый канал общения с поставщиками.

Это первый из четырёх этапов апгрейда модуля. Последующие этапы (обзор по всем
поставщикам; напоминания и сигналы; полировка и скорость) — отдельными спеками.

## Цель этапа

Дать возможность отправлять поставщику Excel-сводку заявок за день **по email** —
вручную кнопкой и автоматически в дедлайн (по желанию, настраивается по поставщику).

## Что уже есть и переиспользуется

- **Серверная сборка Excel-сводки**: `scripts/build_so_order_xlsx.mjs`, вызывается из
  `so/admin/send-summary` (`api/includes/supplier_orders.php`, ~строка 2457+). Тот же
  файл уходит в Telegram — используем его как вложение письма.
- **Почтовая инфраструктура**: `sendEmail()` (`api/includes/mail_send.php`),
  брендированный шаблон `renderMailHtml()` (`api/includes/mail_templates.php`),
  SMTP account=order, Reply-To `order@supply-department.online`.
- **Адреса поставщика**: `suppliers.email` (VARCHAR 255) + `suppliers.cc_emails` (TEXT) —
  те же, что использует отправка заявки из /order. Новое поле почты не нужно.
- **Настройки поставщика**: `so_supplier_settings` (`is_accepting_orders`,
  `auto_submit_previous`, `default_deadline_time`, `pause_message`).
- **Крон дедлайнов**: `api/cron_telegram.php` (~строка 1064+) — после дедлайна для
  поставщиков с `auto_submit_previous=1` подставляет прошлые заявки; защита от дублей
  через `so_auto_submit_log` (INSERT IGNORE по уникальному ключу).

## Функциональные требования

### 1. Ручная отправка (кнопка)
- Кнопка **«Отправить на почту»** в панели закупщика (`SupplierOrdersManagerView.vue`),
  рядом с существующей кнопкой отправки сводки в Telegram (вкладка «Статус», блок
  `rom-export-row`).
- Отправляет сводку за **выбранный день** (`selectedDate`) на почту текущего поставщика.
- Неактивна, если у поставщика не заполнен `email`; тултип-подсказка «У поставщика не
  указана почта».
- Если за день нет заявок — предупреждение «Нет заявок за этот день», письмо не уходит.
- Успех/ошибка — тост.

### 2. Авто-отправка в дедлайн (по поставщику, по умолчанию выкл)
- Новый флаг `so_supplier_settings.auto_email_summary TINYINT(1) DEFAULT 0`.
- Чекбокс «Автоматически отправлять сводку на email в дедлайн» в настройках поставщика
  (вкладка «Графики»/настройки в `SupplierOrdersManagerView.vue`).
- В `cron_telegram.php`, **после** блока авто-подстановки прошлых заявок для поставщика,
  для поставщиков с `auto_email_summary=1` и заполненным `email`: если за прошедший
  по дедлайну день есть хотя бы одна заявка — отправить письмо один раз.
- Защита от дублей: уникальный ключ в `so_email_log` + INSERT IGNORE (аналог
  `so_auto_submit_log`).

### 3. Письмо
- Шаблон `renderMailHtml()` (фирменный, с логотипом и брендовыми цветами).
- Тема: «Заявки на <ДД.ММ> — <Полное имя поставщика>».
- Тело: приветствие + «Направляем заявки ресторанов на <дата>» + строка «Ресторанов: N,
  позиций: M» + «Детали — во вложении (Excel)» + подпись. Таблицу позиций в HTML не
  вставляем (детали в xlsx) — как в отправке плана/заявки.
- Вложение: xlsx от `build_so_order_xlsx.mjs`. Лимит вложения 4 МБ (как в существующих
  отправках).
- Получатели: `suppliers.email` (To) + `suppliers.cc_emails` (CC). Reply-To
  `order@supply-department.online`.

## Данные (БД)

Миграция `migrations/20260715_so_email_summary.sql`:

1. `ALTER TABLE so_supplier_settings ADD COLUMN auto_email_summary TINYINT(1) NOT NULL DEFAULT 0`
   — обратная совместимость: 0 = как раньше (авто-отправки нет).

2. Новая таблица `so_email_log` — журнал отправок (**чистый лог, без уникальных ключей**;
   ручная отправка может повторяться сколько угодно, каждая пишется отдельной строкой):
   - `id` PK
   - `supplier_id` CHAR(36)
   - `delivery_date` DATE
   - `legal_entity` VARCHAR
   - `recipients` TEXT (To через запятую)
   - `cc_recipients` TEXT NULL
   - `subject` VARCHAR(255)
   - `restaurants_count` INT, `items_count` INT
   - `trigger_type` ENUM('manual','auto')
   - `success` TINYINT(1), `error_message` TEXT NULL
   - `sender_user_name` VARCHAR NULL (для manual), `ip_address` VARCHAR NULL
   - `created_at` TIMESTAMP
   - Индексы: (supplier_id, delivery_date), (created_at).

3. Новая таблица-сторож `so_email_auto_log` — защита авто-отправки от дублей, точная
   копия паттерна `so_auto_submit_log`:
   - `supplier_id` CHAR(36), `delivery_date` DATE, `created_at` TIMESTAMP
   - `UNIQUE KEY (supplier_id, delivery_date)`.
   - Крон делает `INSERT IGNORE` перед отправкой; если затронуто 0 строк — уже отправляли,
     выходим без письма. Так авто-письмо уходит ровно один раз на (поставщик, день).

## Бэкенд

Файл `api/includes/supplier_orders.php`:

- **Общая функция** `soBuildAndSendSummaryEmail($pdo, $supplierId, $deliveryDate, $legalEntity, $triggerType, $sender = null): array`
  - Собирает заявки за день (та же выборка, что у send-summary/export).
  - Строит xlsx через существующий путь (node-генератор).
  - Достаёт `suppliers.email` + `cc_emails`, полное имя поставщика.
  - Рендерит письмо `renderMailHtml()`, шлёт `sendEmail()` с вложением.
  - Для `trigger='auto'`: сначала `INSERT IGNORE` в `so_email_auto_log`; если 0 строк —
    уже отправляли, выходим без письма. Затем отправка и запись в `so_email_log`.
  - Для `trigger='manual'`: сразу отправка и запись в `so_email_log` (без сторожа).
  - Возвращает `{ success, sent, error?, restaurants_count, items_count }`.
- **Эндпоинт** `POST so/admin/send-summary-email` — ручная отправка. Проверка прав
  (`requireModuleAccess('supplier-orders', 'edit')` или текущий паттерн модуля) и группы
  юрлица. Принимает `supplier_id`, `delivery_date`. Вызывает общую функцию с
  `trigger='manual'`, передаёт имя отправителя и IP.
- **Настройки** `so/admin/settings` (GET/POST) — добавить чтение/запись
  `auto_email_summary`.

Файл `api/cron_telegram.php`:
- В цикле по поставщикам, после авто-подстановки прошлых заявок, для
  `auto_email_summary=1` и непустого `suppliers.email`: вызвать
  `soBuildAndSendSummaryEmail(..., 'auto', null)`. Порядок гарантирует, что подставленные
  заявки попадут в сводку.

## Крайние случаи и надёжность

- Нет `email` у поставщика → кнопка неактивна; авто-отправка пропускает.
- Нет заявок за день → авто-отправка молчит; ручная предупреждает.
- Повтор крона → нет дублей (сторож `so_email_auto_log` + INSERT IGNORE).
- Ошибка SMTP → запись `success=0` + `error_message`; ручная показывает тост ошибки.
- Вложение > 4 МБ → не отправляем, сообщаем об ошибке (как в существующих отправках).
- Авто-письмо строго после авто-подстановки заявок.

## Проверка (ручная — автотестов в проекте нет)

1. Ручная отправка через интерфейс на тестовый ящик: проверить тему, вид письма и
   вложение Excel.
2. Кнопка неактивна у поставщика без почты.
3. Авто-отправка: на тестовом поставщике включить галочку, задать тестовый `email`,
   прогнать `cron_telegram.php`; проверить письмо и запись в `so_email_log`.
4. Повторный прогон крона — письмо не задваивается.
5. День без заявок — авто-отправки нет.
