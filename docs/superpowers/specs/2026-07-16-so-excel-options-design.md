# Stage 4 — Excel-отчёт заявок: опции (нулевые строки + паллеты/вес)

Дата: 2026-07-16
Статус: утверждён к реализации (приоритет пользователя — начать с Excel)

## Контекст

Excel-сводка заявок (кнопка «Экспорт»/скачивание в панели закупщика) строится
серверным генератором `scripts/build_so_order_xlsx.mjs` из данных
`soBuildSummaryXlsx()` (`api/includes/supplier_orders.php`). Формат — МАТРИЦА:
строки = рестораны (сгруппированы по городам, с подытогами «Итого <город>» и общей
строкой «ИТОГО»), столбцы = товары (SKU+название), в ячейках — количество.

Нужны две ОПЦИОНАЛЬНЫЕ функции отчёта (по умолчанию ВЫКЛ — текущий вид не меняется):
1. Убирать пустые строки — рестораны без заказа.
2. Добавлять сводку по паллетам/весу по каждому товару.

## Решения (согласовано с пользователем)

- Обе функции — **опциональные** (галочки при экспорте, по умолчанию выкл).
- Колонки/метрики паллет-веса (все четыре, включаются вместе):
  - **Место на паллете (доля)** — сколько паллет занимает суммарный заказ товара.
  - **Вес нетто (кг)**, **Вес брутто (кг)** — суммарный вес заказа товара.
  - **Число коробок**.
- Применяется к **скачиваемому** отчёту (экспорт закупщика). Рассылки поставщику
  (Telegram/email) в этом этапе не трогаем — при желании добавим позже.

## Данные (уже есть, миграций НЕ нужно)

`products`: `qty_per_box` (штук в коробке), `boxes_per_pallet` (коробок на паллету),
`weight_netto`, `weight_brutto` (вес ОДНОЙ КОРОБКИ в ГРАММАХ), `unit_of_measure`.
Заполнены у 100% товаров (435/435).

Расчёт по товару (от суммарного количества заказа по столбцу):
- коробки = `qty_в_коробках` (см. ниже про единицу) — если количество в штуках:
  `boxes = qty / qty_per_box`; если уже в коробках — `boxes = qty`.
- место на паллете (доля) = `boxes / boxes_per_pallet`.
- вес нетто, кг = `boxes * weight_netto / 1000`.
- вес брутто, кг = `boxes * weight_brutto / 1000`.

**ВАЖНО (проверить при реализации):** в каких единицах хранится `so_order_items`
количество — штуки или коробки. Определить по коду ввода/отображения заявки
ресторана (не угадывать). От этого зависит формула `boxes`. Значения свериться на
реальном отчёте (пользователь проверит).

## Функциональные требования

### 1. Опция «Убрать пустые строки»
- Флаг `hide_zero_rows`. Когда включён — в матрице не выводить строки ресторанов, у
  которых суммарное количество по всем товарам = 0 (не подал / нулевой заказ).
- Города, где после фильтра не осталось ни одной data-строки, — не выводить (вместе с
  их заголовком и подытогом). Строки «Итого <город>» и «ИТОГО» пересчитываются по
  оставшимся.

### 2. Опция «Паллеты и вес»
- Флаг `show_pallet_weight`. Когда включён — ПОСЛЕ строки «ИТОГО» добавить сводные
  строки, по одной на метрику, заполненные по каждому столбцу-товару:
  - «Коробок»
  - «Место на паллете (паллет)»
  - «Вес нетто, кг»
  - «Вес брутто, кг»
- Значения — из суммарного количества по столбцу (уже считается как grand total) и
  атрибутов товара. Округление: коробки/паллеты — до 2 знаков; кг — до 1 знака (уточнить
  на реальном отчёте, легко поменять).
- Стиль строк — как у «ИТОГО»/подытогов (bold, выделение), чтобы визуально отделялись.

## Бэкенд

`api/includes/supplier_orders.php`:
- `soBuildSummaryXlsx($pdo, $supplierId, $deliveryDate, array $options = [])` — новый
  необязательный параметр `$options` (`hide_zero_rows` bool, `show_pallet_weight` bool);
  по умолчанию пусто → текущее поведение. Существующие вызовы (send-summary,
  send-summary-email, cron) НЕ меняются (получают дефолт).
- При `show_pallet_weight` — обогатить `products_map`/`products` полями `qty_per_box`,
  `boxes_per_pallet`, `weight_netto`, `weight_brutto` через выборку из `products` по
  `sku` (справочник — по ГРУППЕ юрлиц поставщика, как остальные справочники). При
  отсутствии товара в справочнике — поля 0/null (метрики по нему = 0).
- Прокинуть `options` (и обогащённые поля товаров) в `payload` для генератора.
- Эндпоинт `so/admin/export` (GET) — читать query-параметры `hide_zero` и `pallet_weight`
  (0/1), передавать в `soBuildSummaryXlsx` как `$options`.

## Генератор `scripts/build_so_order_xlsx.mjs`

- Принять `options`/новые поля товаров из входного JSON.
- `hide_zero_rows`: при построении data-строк пропускать ресторан с нулевой суммой; не
  выводить пустые города; подытоги/ИТОГО — по оставшимся.
- `show_pallet_weight`: после «ИТОГО» добавить 4 сводные строки (см. выше), считая по
  `col total` и атрибутам товара; стилизовать как итоговые.
- Без флагов — вывод байт-в-байт как сейчас.

## Фронтенд

`src/views/SupplierOrdersManagerView.vue` + `src/stores/supplierOrderStore.js`:
- Рядом с элементами экспорта (вкладка «Статус») — две галочки: «Убрать пустые строки»,
  «Паллеты и вес» (по умолчанию выкл).
- `adminGetExport(supplierId, date, options)` — добавить query `hide_zero`/`pallet_weight`
  по галочкам.

## Крайние случаи

- Обе опции выкл → отчёт идентичен нынешнему (регресс-защита).
- Товар без веса/паллет в справочнике → его метрики = 0, отчёт не падает.
- Все рестораны пустые + hide_zero_rows → матрица только с шапкой и «ИТОГО» (нулевым);
  не падать.
- Единица количества (штуки/коробки) — определить из кода, не угадывать.

## Проверка (ручная — автотестов нет)

1. Экспорт без галочек = как раньше.
2. «Убрать пустые строки» → рестораны без заказа исчезли, итоги сошлись.
3. «Паллеты и вес» → снизу 4 строки, значения бьются с ручным пересчётом на 1-2 товарах.
4. Обе галочки вместе.
5. Рассылки поставщику (TG/email) не изменились.
