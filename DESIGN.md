# DESIGN.md — Дизайн-система bk-calc

Короткий «договор» дизайн-системы. Что использовать, чего не делать.
Полный дизайн-док с обоснованием решений: `~/.gstack/projects/bk-calc/alexey-main-design-20260514-065604.md`.

Канон установлен 2026-05-14 в рамках редизайна модуля «Задачи». Применяется
project-wide для всех новых компонентов.

## Стиль

«Refined Trello» — обновлённый Trello-стиль:

- Тёплая нейтральная палитра (beige), согласована с бренд-оранжевым `#E87A1E`.
- Один акцентный цвет — бренд-оранжевый. Используется для главных действий
  и активных состояний, не как декорация.
- Atlassian-семантика приоритетов (urgent / high / medium / low).
- Палитра меток — 8 фиксированных цветов (`--tk-label-1..8`).
- Шрифты: **Flame** — брендовые заголовки/логотип (подключён через `@font-face`,
  файлы `/Flame-*.otf`); **Plus Jakarta Sans** — тело и интерфейс (`body`,
  токен `--tk-font`); **JetBrains Mono / monospace** — числа, коды, данные.
- Радиусы 4/10/14 + 12 для карточек.

## Источник истины — `src/styles/tokens.css`

Все цвета, размеры, отступы, радиусы, тени, длительности анимаций объявлены
в `src/styles/tokens.css` на `:root`. Импортируется в `src/main.js` один раз
до `app.mount`.

**Категории токенов:**

- `--tk-n-0..900` — нейтральная серая шкала (тёплый beige).
- `--tk-text*`, `--tk-bg-*`, `--tk-border*` — семантические токены.
- `--tk-accent*`, `--tk-violet*` — акценты.
- `--tk-label-1..8` — палитра меток.
- `--tk-prio-{urgent,high,medium,low}-{bg,fg}` — приоритеты.
- `--tk-{success,warning,danger,info}*` — состояния.
- `--tk-s-1..7` — отступы (шкала по 4px).
- `--tk-r-{sm,md,lg,card,pill}` — радиусы.
- `--tk-fz-{xs,sm,md,lg,xl,h1,h2}` — размеры шрифта.
- `--tk-fw-{regular,medium,semibold,bold}` — веса.
- `--tk-lh-{tight,base,loose}` — line-height.
- `--tk-shadow-{card,card-hover,card-drag,column,popover,modal}` — тени.
- `--tk-anim-{fast,,slow}` (`--tk-anim` без суффикса = средний) — длительности.
- `--tk-transition` (140ms ease, backward-compat), `--tk-transition-fast`.
- `--tk-z-{base,sticky,dropdown,popover,modal,toast,tooltip}` — z-index шкала.
- `--tk-focus-ring` — единый focus-ring для всех интерактивных элементов.
- `--tk-touch-min` — `44px` минимум для тач-цели на мобильном.

**Канал документации каждого токена** — короткий комментарий в `tokens.css`
с пояснением «зачем» и «где используется».

## Универсальные оболочки — `src/components/ui/`

Project-wide компоненты дизайн-системы:

- `<UiPopover>` — оболочка поповера/диалога. Шапка / контент / футер через
  слоты. Анимация fade + slide-up 4px.
- `<UiMenu>` — выпадающее меню. Пункты — кнопки с классами
  `.ui-menu__item`, `.ui-menu__item--danger`, `.ui-menu__divider`. Hover,
  focus-visible, disabled — встроенно.
- `<UiEmptyState>` — пустое состояние. Иконка через слот `#icon`,
  заголовок / описание через props, действие через `action-label` + `@action`
  или слот `#action`.
- `<UiSkeleton>` — скелетон-плейсхолдер с shimmer-анимацией. Формы:
  `rect` / `circle` / `pill`. Поддерживает `prefers-reduced-motion`.

Дополнительно — уже существующие в `src/components/ui/`:
`BkIcon`, `BurgerSpinner`, `CommandPalette`, `OfflineIndicator`,
`PeriodPicker`, `ReminderTimesEditor`, `SupplyLogo`, `ToastContainer`.

## Где всё это посмотреть

`/ui-kit` — внутренняя страница (только для admin'а): каталог всех
компонентов и токенов в разных состояниях. Используется при разработке
(визуальный контроль) и для регрессионного QA после правок в дизайн-системе.

## Правила (must)

1. **Никакого хардкода цветов / отступов / радиусов в компонентах.** Только
   переменные `var(--tk-*)`. Если хочется цвет, которого нет в `tokens.css` —
   значит, надо его добавить в `tokens.css` с комментарием, а не вписывать локально.
2. **Никаких локальных деклараций `--tk-*` в компонентах.** Переменные
   объявляются ОДИН раз — в `tokens.css` на `:root`. Раньше были дубли в
   `TasksView.vue` (`.tasks-view`, `.search-modal`) и `TaskCardModal.vue`
   (`.task-sidebar`) — удалены в редизайне 2026-05-14.
3. **Одна цветовая семантика за раз.** Если на карточке/чипе уже есть цветовой
   индикатор (приоритет, срок, метка) — не дополнять его рамкой того же цвета.
   Цвет несёт ОДИН смысл за раз.
4. **Карточка кратко.** Максимум 4 мета-сигнала + до 3 аватаров на канбан-карточке.
   Лишнее — в модалку, либо схлопывается в `+N`.
5. **Шрифты — Flame (бренд-заголовки) и Plus Jakarta Sans (тело).** Брендовый
   текст и заголовки — `'Flame', 'Plus Jakarta Sans', sans-serif`; обычный
   интерфейс и тело — `var(--tk-font)` (Plus Jakarta Sans с системным fallback).
   Не вписывать `system-ui` / `-apple-system` ПЕРВЫМ шрифтом — это «я сдался на
   типографике».
   ⚠️ **Известный нюанс:** сейчас Plus Jakarta Sans фактически не подгружается
   (нет `<link>`/`@font-face`), поэтому тело рендерится системным шрифтом.
   Flame подключён и работает. Если решим включить Plus Jakarta Sans — добавить
   его загрузку (link или self-host) отдельной задачей.
6. **Пустые состояния — это фичи.** Каждый экран, где может быть «нет данных»,
   получает `<UiEmptyState>` с иконкой, заголовком, описанием и (если применимо)
   действием. Не «No items found».
7. **Touch-цели на мобильном — минимум 44×44px.** Если визуально иконка
   маленькая — расширяй область клика через `padding`.
8. **Контраст текста к фону — минимум 4.5:1** (WCAG AA, обычный текст).
   Для крупного текста (≥18px) — ≥3:1. Сверять через
   [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/).
9. **Focus-ring обязателен для всех интерактивных элементов.** Использовать
   `box-shadow: var(--tk-focus-ring)` или унаследованную через состояние `:focus-visible`.

## Правила (must not)

- ❌ Декоративные градиенты и блобы в фоне (не несут смысла).
- ❌ Цветные кружки с иконками в карточках (выглядит как SaaS-стартер из 2018).
- ❌ Центрирование ВСЕГО подряд (хедеры, описания, карточки). Иерархия через
  выравнивание по сетке, а не симметрию.
- ❌ Эмодзи как декорация. Эмодзи только в пользовательском контенте.
- ❌ Полоска приоритета слева у карточки (3px цветная) — была заменена на
  цветной чип в мета-строке. Не возвращать.
- ❌ Дублирующие индикаторы прогресса (прогресс-бар + счётчик в одной карточке) —
  оставлять что-то одно.
- ❌ Сторонние UI-библиотеки (Element Plus, PrimeVue, Vuetify, Vuetify, daisyUI).
  Дизайн-система собственная, не нужны зависимости.
- ❌ Анимации без уважения к `prefers-reduced-motion`. Всегда добавлять fallback
  через `@media (prefers-reduced-motion: reduce) { animation: none; }`.

## Тёмная тема

В текущей итерации НЕ реализована. Структура токенов в `tokens.css` подготовлена
так, чтобы тёмную тему можно было добавить через подмену значений на
`[data-theme="dark"]` без переписывания компонентов. Конкретный план — отдельная
итерация после внедрения light-темы во всех модулях.
